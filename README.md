# __🐧Linux 프로세스 관리 명령어 정리__

## 1. _top_

시스템의 실시간 프로세스 상태를 보여주는 명령어입니다.

CPU, 메모리 사용량 확인

어떤 프로세스가 리소스를 많이 쓰는지 실시간 감시 가능

top

종료하기:
q 입력
___
## 2. _ps_

현재 실행 중인 프로세스를 스냅샷 형태로 출력합니다. (실시간 아님)

자주 쓰는 옵션:

__ps aux__

__a__ : 터미널에 상관없이 모든 사용자 프로세스 출력

__u__ : 사용자 정보 포함

__x__ : 터미널 없이 실행된 프로세스 포함

특정 프로세스 검색:

__ps aux | grep 프로세스이름__
___
## 3. _jobs_

현재 쉘에서 백그라운드로 실행한 작업 목록을 조회하는 명령어입니다.

__jobs__

백그라운드 작업 실행 예:

__sleep 100 &__

백그라운드 작업 다시 foreground로 가져오기:

__fg %1__

백그라운드로 유지하며 실행:

__bg %1__

___
## 4. _kill_

실행 중인 프로세스에게 시그널을 보내 종료 또는 제어하는 명령어입니다.

기본 종료 (SIGTERM):

__kill   PID__


강제 종료 (SIGKILL):

 __kill  -9 PID__


jobs에서 가져온 작업 번호 종료:

__kill   %1__


___



# 명령어 요약표

| 명령어    | 역할                |
| ------ | ----------------- |
| `top`  | 실시간 프로세스 상태 모니터링  |
| `ps`   | 현재 실행 프로세스 목록 출력  |
| `jobs` | 현재 쉘의 백그라운드 작업 확인 |
| `kill` | 프로세스 종료 또는 시그널 전달 |
